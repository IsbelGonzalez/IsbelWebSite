Que es HTML

HiperText Markup Language

Es un lenguaje estructurados de etiquetas (tags) jerarquizadas creador Tim Berner-Lee. Cientifico computacion britanico

Permite distribur de forma ordenada los datos de la paguina con una estructura.

La forma que se expresan las etiquetas es de forma jerarquica. Hay unas dentro de otras y esta jerarquia es importate por que tiene sus relaciones que tambien sirve para el css ademas de la estructura

Los tags de html contienen dentro atributos que permiten configurar las caracteristicas de cada etiqueta.

CASCADA

Es el orden de aplicacion de los estilos creados en css

El orden es de arriba (Menor priorida) hacia abajo (Mayor Prioridad)

Para un elemento pueden existir varios estilos que colicionan por esto es la cascada

Losestilos se combinan tomando en cuenta la prioridad pero se hace una combinacion de los que son diferentes.

  1. Nivel mas bajo. De archivo incluido .css: De menor prioridad y la mejor practica (Amarillo oscuro)
  2. Nivel medio bajo. De cabecera (Azul)
  3. Nivel alto. De etiqueta (Verde)
  4. Nivel mayor. De !importatant: De mayor prioridad (Amarilli Ocuro)

Tambien existe cascada en el mismo dcumento si estan repetidos los estilos

Nivel de mismo documento (Primero rojo, Segundo Azul). Toma en cuenta el ultimo definido y combinando.

COLAPSO DE MARGENES

Si varios margenes relacionados se tocan entre si, se pueden formar un unico margen que toma el valor mayor.

Si uno de los margenes es negativo y otro es positivo se suman algebraicamente y si el resultado es negativo se solapan los elementos.

Exepciones

El colapso no ocurre con elementos que flotan.

El colapso no ocurre con elementos posicionados.

El colapso no ocurre con elementos de inline-block.

El colapso no ocurre con elementos que tengan propiedad overflow configurada con algo difeente a visible.

El colapso no ocurre con elementos que tengan propiedad clear

El colapso no ocurre con elementos root (Elemento raiz) de html
