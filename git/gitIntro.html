GIT

GIT

Sistemas distribuido de control de versiones (Distributed revision control system).

Creado por linus Torvalds (2005) y basado en ramas. Estas ramas pensadas para luego fusionarce y crear el proyecto final creando un version master (version de producion).

Esto es GIT rastrea el trabajo colaborativo en proyectos a travez de un hermoso modelo de arbol distribuido (branch-rama) de la teoria de grafos.

Historia

Antes de Git:

  • CVS: Concurrent Version System (tortuga)
  • SVN: SubVersion

Estructura

Branch - Rama

Permite desviarse de la linea principal de desarrollo.

El archivo HEAD es el que guarda la referencia a la rama actual. /.git/HEAD

Las ramas apuntan a un commit. /.git/refs/heads/nombreRama este archivo contiene el id del commit.

Conflictos

branch rama-1 branch master (cambios mismo archivo misma linea diferentes ramas)

$git marge ramaConflicto --> CONFLICT (

Se guardan los cambios dejando las lineas que el progrmador decida.

se agrega la modificacion $git add .

se hace el commit $git commit

Tratar de usar git status y git log entre cada paso para evitar confucion
