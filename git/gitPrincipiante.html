HEAD (los ultimos commit) ^^(cuantos commits, los dos ultimos tambien se puede usar HEAD~15 los ultimos 15 commits)
Repository $git reset --hard HEAD@{1} En el caso q se devolver un commit anteriormente borrado y se quiera volver a obtener el commit borrado. Usar reflog para buscar la referencia.(HEAD@{1})

Repository rebase $git rebase -i HEAD~4 (cambiar los ultimos 4 commits). opciones (pick: usar commit, reword: editar mensaje, squash: usar este commit combinarlo al anterior, fixup: combinar commit y descartar el mensaje, drop: borrar commit). Como esto cambia los commits va a crear conflictos para subir al remoto (git push --force) no recomendado en master y otros desarrolladores deben ejecutar (git fetch origin y git reset --hard origin/master) para poder descargar los cambios y aplicarlos en sus repositorios locales.

Estados de los archivos (cambios)

Archivo .gitignore

En este archivo se agregar los nombres de los archivos que se quieren ignorar para que el repositorio no los tome en cuenta. Tambein se pueden escribir patrones para ignorar varios archivos.

  • nombreArchivo.ext: Se ignoran todos los archivos con este nombre incluidos los que esten en subcarpetas.
  • /nombreArchivo.ext: Se ignoran solo el archivo a nivel de la raiz.
  • carpeta: Se ignoran la carpeta completa que tenga este nombre y las sub carpetas con el nombre indicado.
  • /carpeta: Se ignoran la carpeta que inicie en la raiz y todas su carpetas internas y sus archivos.
  • *.log: Todos los archivos con extencion .log.
  • #: Signo para crear comentarios.

En gitHub hay plantillas de este tipo de archivo para todos los casos posibles como C# o proyectos especificos. gitHub templates
